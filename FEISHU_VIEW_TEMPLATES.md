# 飞书视图创建清单

这份清单对应当前已经落地的上传结构：

- 本地看板负责实时查看
- 飞书负责留底与复盘
- 每个账号最多采集前 `30` 条作品
- 飞书复盘表当前只保留最近 `14` 天
- 每个账号每天只上传：
  - 点赞前 `10`
  - 评论前 `10`

当前推荐长期保留的飞书表只有 3 张：

- `小红书日历留底`
- `每日点赞复盘`
- `每日评论复盘`

后续新增项目，不再新增新表。  
只需要在现有表上复制视图。

## 项目主入口

长期主入口只保留两个排行榜：

- `项目名-今日点赞榜`
- `项目名-今日评论榜`

这两个视图就是每个项目最核心的飞书入口。

其余视图：

- `项目名-点赞复盘`
- `项目名-评论复盘`
- `项目名-日历`
- `项目名-最新留底`

都作为辅助复盘视图保留，不再作为主入口推荐。

推荐在飞书仪表盘或页面里，只固定展示这两块：

1. `项目名-今日点赞榜`
2. `项目名-今日评论榜`

这样后续新增项目时，默认只复制这两个视图即可。

## 4. 新增项目时怎么做

以后每新增一个项目，只做 4 个动作：

1. 在本地系统里把账号归入新项目
2. 跑一次本地采集
3. 上传飞书
4. 在飞书里复制下面这 4 个视图

推荐最少创建的视图：

- `项目名-日历`
- `项目名-最新留底`
- `项目名-点赞复盘`
- `项目名-评论复盘`

当前已经按这个模板建好的项目：

- `默认项目`
- `东莞`

如果这个项目账号很多，再补：

- `项目名-今日点赞榜`
- `项目名-今日评论榜`

## 5. 推荐排序与筛选规则

统一推荐：

- 日期类视图：
  - `日期` 或 `日历日期` 降序
- 榜单类视图：
  - `账号` 升序
  - `账号内排名` 升序

统一筛选：

- 第一层：`项目 = 指定项目`
- 第二层：按是否需要限制 `日期 = 今天`
- 第三层：需要看单账号时再加 `账号 = 指定账号`

## 6. 不建议再做的事

- 不要按项目继续新建新表
- 不要把不同项目混在一个“总览排行榜视图”里
- 不要把历史全量排行长期堆进飞书
- 不要把飞书当主数据仓库

当前最合理的长期结构就是：

- 本地缓存保留全量
- 飞书只保留近期复盘
- 飞书按项目复制视图，不按项目加表
