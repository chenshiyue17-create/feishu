# 项目新增 SOP

这份 SOP 用于后续新增项目时，按固定动作完成：

- 本地项目新增
- 账号归档
- 本地采集
- 飞书上传
- 飞书视图复制

适用前提：

- 当前 Base 已经固定为：
  - `小红书日历留底`
  - `每日点赞复盘`
  - `每日评论复盘`
- 当前系统已可正常：
  - 更新本地看板
  - 上传当前项目到飞书

## 1. 新增项目前先确认

先确认 3 件事：

1. 小红书登录态可用
2. 本地缓存目录正常
3. 飞书 Base 当前可写

## 2. 本地新增项目

在本地系统里完成：

1. 新建项目名
2. 把账号归入该项目
3. 保存监测清单

命名建议：

- 用业务可长期复用的项目名
- 不要用临时活动名当项目主名

推荐：

- `东莞`
- `杭州`
- `上海内购`

不推荐：

- `3月活动`
- `4月冲刺`

## 3. 跑一次本地采集

新增项目后，先只做本地采集：

1. 打开本地页面
2. 选中新项目
3. 点击 `更新本地看板`

确认结果：

- 项目首页能看到新项目
- 账号列表有数据
- 单账号榜单能打开
- 本地缓存目录生成该项目文件夹

缓存目录结构示例：

- `飞书缓存/项目名/dashboard.json`
- `飞书缓存/项目名/calendar_rows.json`
- `飞书缓存/项目名/ranking_rows.json`

## 4. 上传飞书

采集完成后，再做飞书上传：

1. 选中新项目
2. 点击 `上传当前项目到飞书`

或按需只上传：

- `仅上传日历`
- `仅上传排行榜`

上传成功后，飞书里应该能看到：

- `小红书日历留底` 里有该项目的新记录
- `每日点赞复盘` 里有该项目的新记录
- `每日评论复盘` 里有该项目的新记录

## 5. 复制飞书视图

不要新建新表。  
只复制现有项目视图，然后改筛选条件。

最少复制这 4 个：

1. `默认项目-日历`
2. `默认项目-最新留底`
3. `默认项目-点赞复盘`
4. `默认项目-评论复盘`

如果需要当天榜单，再额外复制：

5. `默认项目-今日点赞榜`
6. `默认项目-今日评论榜`

复制后统一修改：

- 视图名称改成新项目名
- 筛选条件里的 `项目 = 默认项目`
- 改成 `项目 = 新项目名`

## 6. 推荐视图命名

每个项目建议固定 6 个视图：

- `项目名-日历`
- `项目名-最新留底`
- `项目名-今日点赞榜`
- `项目名-点赞复盘`
- `项目名-今日评论榜`
- `项目名-评论复盘`

这样以后不同项目之间命名完全一致。

## 7. 新增项目后的验收

新增项目后，按这个顺序检查：

1. 本地项目首页能看到该项目
2. 本地账号列表能看到项目内账号
3. 本地榜单能切到该项目
4. 飞书日历表里能筛出该项目记录
5. 飞书点赞复盘表里能筛出该项目记录
6. 飞书评论复盘表里能筛出该项目记录
7. 该项目视图能正常打开

## 8. 不要做的事

- 不要为每个项目新建新表
- 不要把项目名写进表名
- 不要继续往飞书灌全历史榜单
- 不要把临时活动当长期项目名

## 9. 长期维护规则

长期固定规则：

- 本地缓存保留全量
- 飞书只保留近期复盘
- 飞书靠视图扩项目，不靠加表
- 每个账号最多采集前 `30` 条作品
- 飞书近 `14` 天复盘
- 每账号每天前 `10` 条点赞、前 `10` 条评论

## 10. 当前推荐操作路径

后续任何新项目，统一按这 3 个动作走：

1. `更新本地看板`
2. `上传当前项目到飞书`
3. `复制项目视图`

如果这 3 步都正常，项目就算正式上线。
